# 出荷業務フロー（現状）

## 概要

出荷日当日に営業マンが自分の受注分を倉庫からピッキングし、トラックへ積み込んで顧客へ配送するまでの業務フローを記載する。

**特徴**:
- 那覇魚類では営業マンが配送担当を兼任している
- 受注分以外に「持ち回り」と呼ばれるサンプル出荷がある

---

## 業務フロー

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   事務員    │    │  営業マン   │    │    顧客     │
│             │    │（配送兼任） │    │             │
└──────┬──────┘    └──────┬──────┘    └──────┬──────┘
       │                  │                  │
       │ 1. 伝票手渡し    │                  │
       │─────────────────>│                  │
       │                  │                  │
       │                  │ 2. 伝票確認      │
       │                  │                  │
       │                  │ 3. ピッキング    │
       │                  │   （倉庫）       │
       │                  │                  │
       │                  │ 4. トラック      │
       │                  │    積み込み      │
       │                  │                  │
       │                  │ 5. 配送・納品    │
       │                  │─────────────────>│
       │                  │                  │
```

---

## 各ステップの詳細

### 1. 伝票受け取り

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 場所 | 社内 |
| 手段 | 事務員から複写式伝票（ターンアラウンド１型）を受け取り |
| 備考 | 自分が受注した分の伝票を受け取る |

### 2. 伝票確認

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 場所 | 社内 |
| 手段 | 伝票の内容を目視確認 |
| 備考 | 出荷商品、数量、配送先などを確認 |

### 3. ピッキング

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 場所 | 倉庫 |
| 手段 | 伝票を見ながら商品を集める |
| 備考 | 自分の受注分を自分でピッキング |

### 4. トラック積み込み

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 場所 | 倉庫・積み込み場 |
| 手段 | ピッキングした商品をトラックへ積み込み |
| 備考 | 配送ルートを考慮した積み込み順序 |

### 5. 配送・納品

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 場所 | 顧客先 |
| 手段 | トラックで配送、顧客へ納品 |
| 備考 | 営業マンが担当顧客へ直接納品 |

---

## 現状の課題

### ピッキングに関する課題

- **先入先出（FIFO）の未徹底**: どのロットから出荷すべきかの指示がない
- **在庫確認の困難**: ピッキング時に在庫がない場合、その場で判明する
- **ピッキングミスのリスク**: 伝票と商品の照合が目視のみ

### 業務効率に関する課題

- **営業マンの業務負荷**: 受注・ピッキング・配送を一人で担当
- **出荷検品の欠如**: ピッキングした商品の検品プロセスが不明確

### 在庫管理に関する課題

- **出荷実績の反映タイミング**: 出荷後に在庫がいつ減算されるか不明確
- **リアルタイム在庫との乖離**: 出荷中の在庫が把握できない

---

## 持ち回り（サンプル出荷）

### 概要

「持ち回り」とは、受注していない商品を営業マンが独自の判断で出庫し、顧客へ持っていく行為を指す。

### 目的

- 新商品の紹介・提案
- 顧客が購入したことのない商品の提案
- その場での販売機会の創出

### パターン

| パターン | 内容 |
|----------|------|
| その場で販売 | 商品を見せて、その場で購入してもらう |
| 預け置き | 顧客の冷凍庫・冷蔵庫へ納入し、試しに使ってもらう。使用分のみ請求し、余った分は後日回収 |

### 現状のフロー

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  営業マン   │    │    顧客     │    │   倉庫      │
└──────┬──────┘    └──────┬──────┘    └──────┬──────┘
       │                  │                  │
       │ 1. 商品選定      │                  │
       │   （独断）       │                  │
       │                  │                  │
       │ 2. 出庫          │                  │
       │──────────────────────────────────────>
       │                  │                  │
       │ 3. 顧客へ提案    │                  │
       │─────────────────>│                  │
       │                  │                  │
       ├──── 販売成立 ────┤                  │
       │   または         │                  │
       ├──── 預け置き ────┤                  │
       │   または         │                  │
       ├──── 売れず ──────┤                  │
       │                  │                  │
       │ 4. 売れ残り回収  │                  │
       │<─────────────────│                  │
       │                  │                  │
       │ 5. 再入庫        │                  │
       │──────────────────────────────────────>
       │                  │                  │
```

### 持ち回りに関する課題（重大）

| 課題 | 詳細 | 影響 |
|------|------|------|
| **履歴の欠如** | 持ち回りの記録が一切残っていない | 何を・いつ・どれだけ持ち出したか追跡不可能 |
| **承認プロセスの欠如** | 営業マン個人の独断で実行可能 | 持ち回りの適正管理ができない |
| **再入庫の追跡不可** | 売れ残りの回収・再入庫が記録されない | 在庫数との乖離が発生 |
| **在庫ずれの原因** | 出庫・再入庫が追えない | 棚卸し時に原因不明の差異が発生 |
| **預け置き管理の欠如** | 顧客先に預けている商品の把握ができない | 回収漏れ、請求漏れのリスク |
