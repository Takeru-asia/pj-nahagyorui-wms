# 受注業務フロー（現状）

## 概要

営業マンが顧客から注文を受け付けてから、出荷業務担当者へ伝票が渡るまでの業務フローを記載する。

---

## 業務フロー

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   顧客      │    │  営業マン   │    │   事務員    │    │ 出荷担当者  │
└──────┬──────┘    └──────┬──────┘    └──────┬──────┘    └──────┬──────┘
       │                  │                  │                  │
       │  1. 注文         │                  │                  │
       │─────────────────>│                  │                  │
       │                  │                  │                  │
       │                  │ 2. ノートに      │                  │
       │                  │    手書き記録    │                  │
       │                  │                  │                  │
       │                  │ 3. ノート確認    │                  │
       │                  │─────────────────>│                  │
       │                  │                  │                  │
       │                  │                  │ 4. 販売管理      │
       │                  │                  │    システム入力  │
       │                  │                  │                  │
       │                  │                  │ 5. 伝票印刷      │
       │                  │                  │                  │
       │                  │                  │ 6. 伝票手渡し    │
       │                  │                  │─────────────────>│
       │                  │                  │                  │
```

---

## 各ステップの詳細

### 1. 注文受付

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 場所 | 顧客先（外出先） |
| 手段 | 口頭、電話、FAX など |
| 備考 | 営業マンが各顧客を担当し、注文を受け付ける |

### 2. ノートへ手書き記録

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 場所 | 社内（帰社後） |
| 手段 | 営業マン個人のノートに手書き |
| 備考 | 営業マンごとに専用のノートを使用 |

### 3. ノート確認

| 項目 | 内容 |
|------|------|
| 担当者 | 事務員 |
| 場所 | 社内 |
| 手段 | 営業マンのノートを目視で確認 |
| 備考 | 受注内容の読み取り、不明点があれば営業マンに確認 |

### 4. 販売管理システム入力

| 項目 | 内容 |
|------|------|
| 担当者 | 事務員 |
| 場所 | 社内 |
| 手段 | レガシー販売管理システムへ手入力 |
| 備考 | 商品コードを手動で入力 |

### 5. 伝票印刷

| 項目 | 内容 |
|------|------|
| 担当者 | 事務員 |
| 場所 | 社内 |
| 手段 | 販売管理システムから印刷 |
| 出力物 | 複写式伝票（ターンアラウンド１型） |

### 6. 伝票手渡し

| 項目 | 内容 |
|------|------|
| 担当者 | 事務員 → 出荷業務担当者 |
| 場所 | 社内 |
| 手段 | 物理的な伝票の手渡し |
| 備考 | - |

---

## 現状の課題

### コミュニケーションに関する課題

- **手書きノートの読み取りにくさ**: 営業マンの筆跡によっては事務員が読み取りづらい
- **口頭伝達への依存**: 不明点があれば口頭で確認が必要
- **情報の属人化**: 営業マン個人のノートに情報が分散

### データ入力に関する課題

- **二重入力**: 営業マンがノートに記録 → 事務員がシステムに入力という二重作業
- **入力ミスのリスク**: 手書き内容を別の担当者が入力するため、転記ミスが発生しやすい
- **商品コードの手入力**: 手動入力による打ち間違えリスク

### 業務効率に関する課題

- **リアルタイム性の欠如**: 営業マンが帰社するまで受注情報が社内に共有されない
- **物理的な伝票のやり取り**: 伝票の手渡しによる時間ロス
