# 入庫業務フロー（現状）

## 概要

仕入れ先への発注から商品が納入され、販売管理システムへ登録されるまでの業務フローを記載する。

**特徴**:
- 仕入れ先への発注も営業マンが担当している
- 発注情報が社内で共有されていない

---

## 業務フロー

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  営業マン   │    │  仕入れ先   │    │ 検品担当者  │    │   事務員    │
└──────┬──────┘    └──────┬──────┘    └──────┬──────┘    └──────┬──────┘
       │                  │                  │                  │
       │ 1. 発注          │                  │                  │
       │  （電話等）      │                  │                  │
       │─────────────────>│                  │                  │
       │                  │                  │                  │
       │    ※発注内容は社内に共有されない    │                  │
       │                  │                  │                  │
       │                  │ 2. 納品          │                  │
       │                  │─────────────────>│                  │
       │                  │   （納品書付き） │                  │
       │                  │                  │                  │
       │                  │                  │ 3. 納品受取      │
       │                  │                  │    検品          │
       │                  │                  │                  │
       │                  │                  │ 4. 納品書を      │
       │                  │                  │    事務員へ      │
       │                  │                  │─────────────────>│
       │                  │                  │                  │
       │                  │                  │                  │ 5. 販売管理
       │                  │                  │                  │    システム入力
       │                  │                  │                  │
       │                  │                  │                  │ 6. 請求書と
       │                  │                  │                  │    突き合わせ
       │                  │                  │                  │   （後日）
       │                  │                  │                  │
```

---

## 各ステップの詳細

### 1. 発注

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 場所 | 社内または外出先 |
| 手段 | 電話など履歴の残らない方法 |
| 備考 | 発注内容は記録されず、社内に共有されない |

### 2. 納品

| 項目 | 内容 |
|------|------|
| 担当者 | 仕入れ先 |
| 場所 | 倉庫・納品場所 |
| 手段 | トラック等での配送 |
| 備考 | 納品書が添付される |

### 3. 納品受取・検品

| 項目 | 内容 |
|------|------|
| 担当者 | 納入検品担当者 |
| 場所 | 倉庫・納品場所 |
| 手段 | 目視確認 |
| 備考 | 発注内容を知らないため、納品書との照合のみ |

### 4. 納品書の引き渡し

| 項目 | 内容 |
|------|------|
| 担当者 | 検品担当者 → 事務員 |
| 場所 | 社内 |
| 手段 | 納品書の手渡し |
| 備考 | 事務員はこの時点で初めて発注の事実を知る |

### 5. 販売管理システム入力

| 項目 | 内容 |
|------|------|
| 担当者 | 事務員 |
| 場所 | 社内 |
| 手段 | 納品書を見ながら販売管理システムへ入力 |
| 備考 | 仕入れ情報の登録 |

### 6. 請求書との突き合わせ

| 項目 | 内容 |
|------|------|
| 担当者 | 事務員 |
| 場所 | 社内 |
| 手段 | 後日届く請求書と入力内容を照合 |
| 備考 | 納品書と請求書の内容確認 |

## 現状の課題

### 発注管理に関する課題（重大）

| 課題 | 詳細 | 影響 |
|------|------|------|
| **発注履歴の欠如** | 電話等で発注するため記録が残らない | 何を・いつ・どれだけ発注したか追跡不可能 |
| **発注情報の非共有** | 発注内容が社内の誰にも共有されない | 検品担当者・事務員が発注内容を把握できない |
| **発注と納品の照合不可** | 発注内容が不明なため照合できない | 発注通りの納品が行われているか確認できない |

### 検品に関する課題

| 課題 | 詳細 | 影響 |
|------|------|------|
| **検品基準の不在** | 発注情報がないため納品書との照合のみ | 数量過不足・誤納品を検知できない |
| **品質検品の状況不明** | 商品品質の確認プロセスが不明確 | 品質問題の見落としリスク |

### 情報伝達に関する課題

| 課題 | 詳細 | 影響 |
|------|------|------|
| **事後的な情報把握** | 納品されて初めて発注を知る | 入庫準備・在庫計画ができない |
| **請求書照合の限界** | 発注情報がないため、請求内容の妥当性を確認しづらい | 過請求等の見落としリスク |

### 在庫管理に関する課題

| 課題 | 詳細 | 影響 |
|------|------|------|
| **入庫タイミングの遅延** | 納品書→事務員→システム入力の流れで遅延 | リアルタイム在庫との乖離 |
| **ロット管理の状況不明** | 入庫時のロット情報管理が不明確 | 先入先出（FIFO）の実現が困難 |

### 当日出荷（通過在庫）に関する課題（重大）

発注した商品が納品当日に出荷されるケースがあり、在庫計算の整合性が崩れている。

```
【正常な処理順序】
  入庫処理 → 在庫増加 → 出荷処理 → 在庫減少

【実際に発生している処理順序】
  商品納品 → 即出荷 → 出荷処理（在庫マイナス） → 納品書が事務員へ → 入庫処理（在庫増加）
```

| 課題 | 詳細 | 影響 |
|------|------|------|
| **処理順序の逆転** | 出荷処理が入庫処理より先に行われる | 在庫計算の整合性が崩れる |
| **マイナス在庫の発生** | 存在しない在庫から出荷処理される | システム上の在庫がマイナスになる |
| **在庫数量の信頼性低下** | 実在庫とシステム在庫が乖離 | 在庫数量を信頼できない |
| **監査・追跡の困難** | 入出庫の時系列が実態と合わない | 在庫移動の正確な追跡が不可能 |

## 再入庫フロー

通常の仕入れ入庫以外に、以下のケースで再入庫が発生する。

### 1. 持ち回り再入庫

営業マンが持ち回りで持ち出した商品が売れなかった場合の再入庫。

| 項目 | 内容 |
|------|------|
| 担当者 | 営業マン |
| 手段 | 営業マンが自分で倉庫に戻す |
| 記録 | なし（持ち回り自体の記録がないため） |

### 2. 顧客返品による再入庫

顧客から返品された商品の対応フロー。返品理由によって処理が分岐する。

#### 返品理由

| 理由 | 内容 |
|------|------|
| 顧客の発注ミス | 顧客側の注文間違い |
| 那覇魚類の受注処理ミス | 社内での受注処理・入力ミス |
| 商品状態 | 商品の品質・状態に問題がある |

#### 返品対応フロー

```
                        ┌─────────────┐
                        │  顧客返品   │
                        └──────┬──────┘
                               │
                   ┌───────────┴───────────┐
                   │                       │
          ┌────────▼────────┐     ┌────────▼────────┐
          │  受発注ミス     │     │   商品状態      │
          │（顧客/那覇魚類）│     │   による返品    │
          └────────┬────────┘     └────────┬────────┘
                   │                       │
          ┌────────▼────────┐     ┌────────▼────────┐
          │ 再販可能か確認  │     │ 加工品に       │
          └────────┬────────┘     │ 使えるか？     │
                   │              └────────┬────────┘
         ┌─────────┴─────────┐             │
         │                   │    ┌────────┴────────┐
    ┌────▼────┐         ┌────▼────┐        │         │
    │再販可能 │         │再販不可 │   ┌────▼────┐ ┌──▼───┐
    └────┬────┘         └────┬────┘   │使用可能 │ │不可  │
         │                   │        └────┬────┘ └──┬───┘
    ┌────▼────┐         ┌────▼────┐        │         │
    │ 再入庫  │         │加工材料 │   ┌────▼────┐ ┌──▼───────┐
    └─────────┘         │または   │   │加工材料 │ │仕入れ先へ │
                        │  廃棄   │   └─────────┘ │  返品     │
                        └─────────┘               └───────────┘
```

#### 各処理の詳細

| 処理 | 条件 | 内容 |
|------|------|------|
| 再入庫 | 受発注ミスで再販可能な状態 | そのまま在庫として再入庫 |
| 加工材料 | 再販不可だが加工品に使用可能 | 加工品の材料として使用 |
| 廃棄 | 再販不可かつ加工にも使用不可 | 廃棄処理 |
| 仕入れ先返品 | 商品状態不良で加工にも使用不可 | 仕入れ先に連絡し返品 |

### 再入庫に関する課題

| 課題 | 詳細 | 影響 |
|------|------|------|
| **持ち回り再入庫の記録なし** | 何を戻したか記録されない | 在庫数との乖離、追跡不可能 |
| **返品処理の記録不明** | 返品理由・処理結果の記録状況が不明確 | 返品傾向の分析ができない |
| **初回検品の欠如** | 納入時に検品していない | 出荷後に顧客から商品状態の問題が発覚 |
| **品質問題の後追い発覚** | 顧客確認後に初めて品質問題が判明 | 顧客満足度低下、返品コスト増加 |
