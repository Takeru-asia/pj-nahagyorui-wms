## 🎨 図の表記ルール

### 色分けの意味

- 🔴 **赤色（濃）**: 重大な問題（記録なし、追跡不可能、経営に直結）
- 🟠 **赤色（薄）**: 改善が必要な問題（業務効率、品質に影響）
- 🟢 **緑色**: 改善後の状態、新システム、ポジティブな効果
- 🔵 **青色**: 中間的な状態、連携、データフロー
- 🟡 **黄色**: 注意事項、特記事項
- 🟣 **紫色**: 原価・コスト関連

### 線の表記

- **実線 (→)**: 通常のフロー、確立された処理
- **点線 (-.->)**: 記録されない・不明確なフロー、問題のある処理
- **太線 (==>)**: 重要な流れ、強調したいフロー

## 📖 図の読み方

### Mermaid記法について

これらの図解は **Mermaid** という記法で作成されています。

- **GitHub**: ファイルを開くと自動的に図が表示されます
- **VSCode**: Mermaid プレビュー拡張機能をインストールすると表示できます
- **その他**: [Mermaid Live Editor](https://mermaid.live/) でコードをコピペして表示できます

### 図の種類

| 種類 | 用途 |
|------|------|
| `graph TB/LR` | フローチャート、プロセス図 |
| `sequenceDiagram` | シーケンス図、時系列の処理 |
| `stateDiagram` | 状態遷移図 |
| `erDiagram` | データモデル、ER図 |
| `mindmap` | マインドマップ、階層構造 |
| `quadrantChart` | 優先度マトリクス |
